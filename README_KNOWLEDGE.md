# 🧠 佐迪网站项目知识库

欢迎来到佐迪智能家具官网项目的知识库！本知识库采用Karpathy LLM Wiki模式设计，旨在积累、组织和传承项目知识。

## 📁 知识库结构

### 1. 核心知识 (knowledge/)
- **project/**: 项目核心知识（愿景、架构、技术栈等）
- **domain/**: 领域知识（智能家具行业、市场分析等）
- **operations/**: 运维知识（部署、监控、故障排除等）

### 2. 过程知识 (process/)
- **development/**: 开发流程（工作流、代码审查、测试等）
- **deployment/**: 部署流程（CDN配置、自动化部署等）
- **maintenance/**: 维护流程（日常检查、性能审计等）

### 3. 技术知识 (technical/)
- **frontend/**: 前端技术（Vite配置、React模式等）
- **backend/**: 后端技术（API设计、数据库等）
- **infrastructure/**: 基础设施（CDN配置、监控工具等）

### 4. 决策日志 (decisions/)
记录所有重要技术决策，包括决策背景、选项分析、最终选择和结果评估。

### 5. 性能指标 (metrics/)
跟踪项目的各项性能指标，包括构建指标、运行时指标和业务指标。

## 🚀 快速开始

### 查找知识
```bash
# 使用搜索工具查找相关知识
./scripts/knowledge/search.sh "关键词"

# 查看知识库索引
cat knowledge/INDEX.md
```

### 添加知识
1. 确定知识所属的类别
2. 使用对应的模板创建文档
3. 遵循文档规范编写内容
4. 更新相关索引文件

### 更新知识
1. 检查知识是否过期
2. 更新文档内容
3. 更新版本信息和日期
4. 通知相关团队成员

## 📝 文档规范

### 文件命名
- 使用小写字母和连字符：`vite-config-optimization.md`
- 包含日期：`2024-04-12-cdn-decision.md`
- 描述性名称：`performance-monitoring-setup.md`

### 文档结构
```markdown
# 文档标题

## 概述
简要描述文档内容

## 背景
为什么需要这个知识

## 详细内容
知识的具体内容

## 相关链接
链接到其他相关知识

## 更新历史
- 2024-04-12: 创建文档
- 2024-04-13: 更新配置信息

## 维护者
文档的维护者信息
```

### 质量标准
- ✅ 准确性：信息准确无误
- ✅ 完整性：内容完整无缺失
- ✅ 时效性：及时更新最新信息
- ✅ 可读性：易于理解和阅读

```bash
./scripts/knowledge/search.sh --tag=performance "优化"
```
