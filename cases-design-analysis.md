# 欧林网站案例展示风格分析

## 案例页面结构

### 1. 案例分类导航
- 按行业/应用场景分类：央企丨国企、外资企业、科技丨服务、医院丨养老、金融机构丨律所、教育丨公寓
- 采用水平导航栏，用户可快速切换不同分类

### 2. 案例展示方式
- **列表展示**：每个案例显示为一个卡片/块
- **包含内容**：
  - 案例名称（如"中国移动"）
  - 项目产品列表（使用的产品名称）
  - 客户简介（文字描述，通常是企业介绍）
  - 可能包含案例图片（背景图）

### 3. 视觉设计特点
- 深色背景（黑色/深灰色）配白色文字
- 案例卡片有图片背景
- 简洁的排版，重点突出案例名称和产品信息
- 有"了解更多"链接，可能跳转到详情页

## 佐迪网站案例系统设计建议

### 数据库表结构
```
cases 表：
- id (主键)
- title (案例标题，如"某某企业办公空间改造")
- category (分类，如"办公空间"、"教育空间"等)
- description (案例描述)
- image (案例主图)
- images (案例图片列表，JSON格式)
- products (使用的产品列表，JSON格式)
- location (项目地点)
- completedDate (完成日期)
- isActive (是否发布)
- createdAt (创建时间)
- updatedAt (更新时间)
```

### 前端展示
- 案例列表页：按分类展示案例卡片
- 案例详情页：展示完整的案例信息和图片轮播
- 案例分类导航：快速切换不同分类

### 后台管理
- 案例管理：创建、编辑、删除案例
- 图片上传：支持多张图片上传
- 分类管理：管理案例分类
